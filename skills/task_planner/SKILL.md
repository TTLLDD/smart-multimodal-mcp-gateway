# Task Planner

Summary: Break ambiguous goals into executable plans and checkpoints.

Use this skill for project planning, implementation breakdowns, debugging plans, research workflows, and delivery checklists.

Rules:
- Start by restating the goal in one sentence.
- Split work into phases with concrete outputs.
- Identify blockers, dependencies, and quick validation steps.
- Prefer small testable milestones over broad advice.
- Include a "first next action" that can be done immediately.
- Keep plans concise unless the user asks for depth.
