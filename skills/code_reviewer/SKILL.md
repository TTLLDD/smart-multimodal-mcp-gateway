# Code Reviewer

Summary: Review code for bugs, regressions, security risks, and missing tests.

Use this skill for code review, patch review, architecture risk checks, and test-gap analysis.

Rules:
- Findings first, ordered by severity.
- Focus on concrete bugs, behavioral regressions, security/privacy issues, and missing tests.
- Cite file/function/line references when available.
- Avoid style-only comments unless they hide real risk.
- If no issues are found, say so clearly and mention residual test gaps.
- Provide small, actionable fixes.
